xquake - quake fo' xos

jack palevich

introduction

xquake be a version o' id softwarez classic first-person-shootin' game "quake". Xquake has been adapted ta run on da xbox development kit, unda da xbox operatin' system.

build instructions

xquake can be built ta run on eitha tha xbox os or on win32.

build instructions fo' xbox os.

  1. install da xos build environment.
  2. build da xquake program.
  3. copy tha xquake program ta tha xbox hard disk.
  4. copy tha medizzle folda ta tha same directory on yo' goddamn xbox hard disk.
  5. run xquake.
     
    herez da sequence o' razzle shell commands dat do steps 2 thru 5:
     

    debug build:

    test games\xquake
    bcz
    xbcp -y \xboxbins\dump\xquaked.xbe xe:\
    xbcp -r -t medizzle xe:\
    xbreboot xe:\xquaked.xbe

    release build:

    set ntdebug=ntsdnodbg
    test games\xquake
    bcz
    xbcp -y \xboxbins\dump\xquake.xbe xe:\
    xbcp -r -t medizzle xe:\
    xbreboot xe:\xquake.xbe

    link-time-code-generation release build:

    set ntdebug=ntsdnodbg
    private
    bcz ltcg
    test games\xquake
    xbcp -y \xboxbins\dump\xquakeltcg.xbe xe:\
    xbcp -r -t medizzle xe:\
    xbreboot xe:\xquakeltcg.xbe

     

    bonus features:

    you can define xquake_wma n'/or xquake_dmusic in yo' goddamn build environment before buildin' ta turn on wma n' dmusic support. If yo' do dis, yo' have ta manually copy tha data files ta tha right place on yo' goddamn hard disk. Read da source code fo' details.

build instructions fo' win32

  1. install visual studio 6.
  2. install tha directx 8 sdk. (currently dx8 be in beta. Microsoft employees can access recent beta builds from \\dxrelease\dx8_ext
  3. open tha win32\win32.dsp project file.
  4. choose tha menu item "build|set active configuration..."
    1. choose tha "win32 debug" configuration.
    2. click ok ta dismiss tha dialog.
  5. choose da menu item "project|settings..."
    1. click on tha "debug" tab
    2. enta "..\media\quake" in da "workin' directory" text field.
    3. click ok ta dismiss tha dialog.
  6. build n' run tha project (f7, f5) as aiiight.

cheat codes

what would a porno game be without cheat codes? a cheat code be a sequence o' button presses dat do somethang special. I've extended da bind command ta allow cheat codes ta be defined. Tha syntax be:

bind $key,key,key... command

thatz a dolla sign, followed by a comma-separated list o' keys.

the default cheat codes r' as follows. Y = press y button, a = press a button, etc.

source code status

xquake contains code n' data from several sources:

  1. the glquake code dat id software has published n' made freely available unda tha gnu public license.
  2. jack palevichz d3dquake, which be a public domain port o' quake ta d3d. peep http://www.palevich.com/3d/d3dquake
  3. the data files from tha free demo version o' quake.
  4. xbox specific code, which be da property o' microsoft.

 xquake gotta also work wit' tha full quake data files. Simply purchase a copy o' quake, n' copy da pak.Pak file from tha quake cd ta c:\tdata\{00000~1\media\quake\id1 directory on yo' goddamn xbox.
